# VERIFIER AGENT — charter and verification protocol

Rewritten 2026-07-20. Audience: a verifier agent auditing the main agent's
claims, or any agent that wants to check its own work before reporting.

**This file contains no project state.** That is deliberate. The previous
version of this file was a state ledger — it restated HEAD commit, tree
cleanliness, clip counts, and "actions taken" — and within one day four of
those claims were false while reading as authoritative. State lives in
`HANDOFF_SESSION_*.md` (newest wins) and is regenerated by the commands
below. This file holds only the *procedures*.

---

## 0. Prime directive

**Every number in a report must be traceable to tool output produced in the
same session.** Not to a handoff doc, not to memory, not to a previous
agent's summary. Handoff docs are hypotheses about the past; only commands
produce facts about the present.

Corollary: when a doc and the repo disagree, the repo wins and the doc gets
fixed. Do not silently reconcile — say which was wrong.

## 1. What this project has been burned by

Read these before auditing anything; they are the failure shapes that recur.

1. **Same-corpus benchmarks that flatter.** Qmax scored 63–67% on internal
   full recordings and **0%** on wild 60 s clips. The binding project rule:
   *the wild set in `~/sung_tests` is the only scoreboard, and no
   same-corpus benchmark green-lights anything.* Applies to eval-set
   *construction* too, not just eval results — see §3.4.
2. **Confident restatement of stale numbers.** The 63-clip scoreboard
   (8/36 top-1) got quoted after the set grew to 106. Different denominator,
   not comparable. Always check the set size the number was computed on.
3. **Docs asserting work that was never done.** The prior version of this
   file claimed the transcript cache had been stripped from tracking. It had
   not (and still is tracked). Verify "action taken" claims with git, always.
4. **Fluent hallucination under stress.** Degraded audio pushes whisper into
   confident wrong output that *defeats* the usability gates. A high
   confidence score is not evidence of correctness anywhere in this system.

## 2. Standing constraints to enforce

- **Zero-cost window** (~30 days from 2026-07-18). No paid API calls. The
  meanings batch is staged and gated on `CARNATIFY_BILLING_OK=1`; Deepti
  **explicitly declined** authorizing it on 2026-07-19. An agent proposing
  it as an "immediate next step" is a red flag — the prior version of this
  file did exactly that.
- **Deploys are Deepti-run.** Agent deploys are blocked by the permission
  classifier. A report claiming a deploy happened is wrong.
- **Pushes are Deepti-run.** `git log origin/main..HEAD` is routinely
  non-empty; that is expected, not a problem to fix.
- **Production model files** (`models/raga_classifier.pkl`) need explicit
  sign-off before being overwritten.

## 3. Verification procedures

Run the command; quote the output. Never paraphrase a number.

### 3.1 Repo state

```bash
git log --oneline -1                      # actual HEAD
git log --oneline origin/main..HEAD       # unpushed (non-empty is normal)
git status --porcelain                    # tree state; "clean" is a claim to check
git ls-files --error-unmatch <path>       # is this file actually tracked?
```
Claims to distrust until run: "tree is clean", "committed", "stripped from
tracking", "gitignored", any specific commit hash.

### 3.2 The scoreboard

```bash
caffeinate -i venv_train/bin/python identify_clip.py ~/sung_tests --no-raga
# drop --no-raga for raga numbers (loads the 1.1 GB RF, much slower)
```
The script prints per-clip `TRUTH` lines and a final `SCORE` block.
**Quote the SCORE block verbatim. Never recompute by hand, never average
sub-slices.**

Check the denominator matches the current set:
```bash
ls ~/sung_tests/*.m4a ~/sung_tests/*.mp3 2>/dev/null | wc -l   # total clips
ls ~/sung_tests/ | grep -c "__OOC"                             # must-abstain
```
If a report's denominators don't match these, the number is stale.

Runtime honesty: "minutes" only holds with a warm transcript cache. Check
coverage before promising a duration:
```bash
python3 -c "
import json,os,unicodedata
d=os.path.expanduser('~/sung_tests')
clips=[f for f in os.listdir(d) if f.lower().endswith(('.m4a','.mp3','.wav'))]
c=json.load(open('data/whisper_transcripts_turbo_stems.json'))
k={unicodedata.normalize('NFC',x) for x in c}
miss=[f for f in clips if unicodedata.normalize('NFC',f) not in k]
print(f'{len(clips)} clips, {len(miss)} need full demucs+whisper')"
```
Each uncached clip is demucs + ~7 whisper passes on CPU — hours, not minutes.

### 3.3 Out-of-catalog claims

Never accept "this clip is out of catalog" without:
```bash
grep -i "<title fragment>" data/catalog_titles.txt
```
7 of 10 hand-picked "OOC" clips were actually in the 8,688-entry registry.
Also: registry raga metadata is not trustworthy — the registry contains rows
parsed from addresses and dates. Raga truth needs a raga-vocabulary check or
Deepti's ear.

### 3.4 Any ML training claim (the one that needs the most scrutiny)

Before believing a training metric, audit the **evaluation set construction**,
not just the score. The question is always: *what is the cheapest way to get
this number that isn't the capability we care about?*

For retrieval/CSI work specifically — check whether a query can be answered
by matching a different crop or augmentation of the **same source
recording**. That is near-fingerprinting, not composition-level ID, and it
is the exact capability that already failed at 0% on wild clips.

```bash
# trace every val clip back to its true source recording; count how many have
# a same-recording sibling sitting in the reference pool
python3 - <<'EOF'
import json, collections
rows=[json.loads(l) for l in open('data/coverhunter_data/val.jsonl')]
by=collections.defaultdict(list)
for r in rows: by[r['song_id']].append(r)
print("val clips:", len(rows), "songs:", len(by))
EOF
```
Precedent (2026-07-20 audit): dev `hit_rate` reached 1.0 and read as success,
but 142 of 189 val clips (75%) had a same-recording sibling in the ref pool
— `real0`/`real1` are un-augmented crops of the *same mp3s* the `aug_` clips
were cut from. The metric could not distinguish the capability from its
trivial substitute. Saturated metrics deserve suspicion, not celebration.

Also check: is the val split disjoint at the level that matters (composition,
*and* source recording)? Is the split keyed on a field that survives the
pipeline? (`tools.extract_csi_features` rewrites utt ids — splitting on utt
silently produced an empty dev set once.)

### 3.5 Environment claims

Two venvs, not interchangeable: `venv` (py3.14 — sklearn, librosa, mirdata,
anthropic) and `venv_train` (py3.11 — demucs, whisper, essentia, rapidfuzz).
Matcher/ASR/eval run in **venv_train**. No ffmpeg on the machine: audio loads
via librosa/essentia, whisper is fed numpy arrays, never file paths.

## 4. Audit checklist for a main-agent report

- [ ] Every number traceable to in-session tool output?
- [ ] Scoreboard quoted as a verbatim SCORE block, denominators current?
- [ ] Any metric compared across different eval-set sizes?
- [ ] "Committed" / "clean" / "stripped" claims checked against git?
- [ ] Any benchmark that is same-corpus, or any eval set with a trivial
      shortcut to the target metric?
- [ ] Any saturated metric (100%, 1.0) treated as success without asking
      what else could produce it?
- [ ] Any proposed step that spends money, deploys, pushes, or overwrites a
      production model?
- [ ] Failures reported as plainly as successes? A report with no negative
      results from a long session is itself suspicious.
- [ ] Claims about file contents made without reading the file?

## 5. How to report

State what you verified, what you could not verify, and what was wrong —
in that order. Quote the command and its output for anything contested.
"Could not verify" is a valid and useful finding; guessing is not.

Do not fix the main agent's work silently. Report the discrepancy and let
Deepti decide, especially where a musical judgment (raga truth, meaning
quality) is involved — she is the domain authority, not the agent.

## 6. Ops traps that waste verifier time

- `zsh` eats unquoted `===` and leading-`=` args — quote them.
- macOS filenames are NFD; transcript cache keys are NFD. A shell-typed NFC
  path silently misses the cache and re-runs ASR.
- Closing the MacBook lid suspends background runs. `caffeinate -i` blocks
  idle sleep but **not** clamshell sleep — lid stays open for long runs.
- Heavy evals and heavy extraction contend for CPU; demucs times out under
  load. Check `uptime` before starting a second heavy job.
- Grepping a dead process's log shows the same line forever. Confirm the PID
  is alive before concluding "no progress."
- Background bash tasks outlive the nominal 600 s timeout; don't restructure
  work around that cap.
